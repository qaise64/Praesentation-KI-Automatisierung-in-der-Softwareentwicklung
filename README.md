# Praesentation: KI-Automatisierung in der Softwareentwicklung

Hallo! Dies ist eine Praesentation, die ich im Juni 2025 an der Hochschule Niederrhein gehalten habe. Das Thema ist der Einfluss von Kuenstlicher Intelligenz auf den *gesamten* Software Development Lifecycle (SDLC).

[**Hier geht's direkt zur vollstaendigen Praesentation (PDF) – sie oeffnet im Browser.**](./SoftwareDevelopment_automation.pdf)

## Worum geht es?

Die Kernidee ist nicht, dass KI Entwickler "ersetzt", sondern als ein **Augmentation**-Werkzeug (also als Erweiterung) dient.

Die Praesentation zeigt, wie KI ueber die reine Code-Generierung hinausgeht und die einzelnen Phasen eines Softwareprojekts beeinflusst.

### Behandelte Themen:

* **Mehr als nur Coden:** Der Einsatz von KI in *allen* Phasen – von der Anforderungserfassung (Copilot4DevOps) ueber Design (Auto-Mockups) bis zu automatisierten Tests und vorausschauender Wartung.
* **Die Werkzeuge:** Ein Blick auf Tools wie GitHub Copilot, aber auch auf die Rolle von KI in CI/CD-Pipelines, bei der Echtzeit-Fehlererkennung und der Testfall-Generierung.
* **Produktivitaet vs. Akzeptanz:** Eine Uebersicht von Industriedaten, die zeigt, *wo* KI den groessten Produktivitaetsschub bringt (Implementierung) und *wo* sie derzeit am staerksten angenommen wird (Anforderungen).
