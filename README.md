# Praesentation: KI-Automatisierung in der Softwareentwicklung

Hallo! Dies ist eine Praesentation, die ich im Juni 2025 an der Hochschule Niederrhein gehalten habe. Es geht um ein Thema, das mich persoenlich sehr interessiert: den Einfluss von Kuenstlicher Intelligenz auf den *gesamten* Software Development Lifecycle (SDLC).

[**Hier geht's direkt zur vollstaendigen Praesentation (PDF) – sie oeffnet im Browser.**](./SoftwareDevelopment_automation.pdf)

## Worum geht es?

Die Kernidee ist nicht, dass KI Entwickler "ersetzt", sondern als ein maechtiges **Augmentation**-Werkzeug (also als Erweiterung) dient.

In dieser Praesentation habe ich analysiert, wie KI *ueber* die reine Code-Generierung hinausgeht und jede einzelne Phase eines modernen Softwareprojekts beeinflusst.

### Was diese Praesentation beleuchtet:

* **Mehr als nur Coden:** Wie KI bereits in *allen* Phasen eingesetzt wird – von der Anforderungserfassung (Copilot4DevOps) ueber Design (Auto-Mockups) bis hin zu automatisierten Tests und vorausschauender Wartung.
* **Die Werkzeuge:** Ein Blick auf Tools wie GitHub Copilot, aber auch auf die Rolle von KI in CI/CD-Pipelines, bei der Echtzeit-Fehlererkennung und der intelligenten Testfall-Generierung.
* **Produktivitaet vs. Akzeptanz:** Eine Analyse von Industriedaten, die zeigt, *wo* KI den groessten Produktivitaetsschub bringt (Implementierung) und *wo* sie derzeit am staerksten angenommen wird (Anforderungen).
